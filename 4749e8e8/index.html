一.VS详解

1.配置路由重写

默认将所有 HTTP 流量路由到标签为“版本：v1”的评论服务的 pod。此外，路径以 /wpcatalog/ 或 /consumercatalog/ 开头的 HTTP 请求将被重写为 /newcatalog 并发送到标签为“version: v2”的 pod

apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: reviews-route
spec:
  hosts:
  - reviews.prod.svc.cluster.local
  http:
  - name: "reviews-v2-routes"
    match:                       #匹配下面两个路径
    - uri:
        prefix: "/wpcatalog"
    - uri:
        prefix: "/consumercatalog"
    rewrite:
      uri: "/newcatalog"        #转发到v2版本的pod的这个路径
    route:
    - destination:
        host: reviews.prod.svc.cluster.local
        subset: v2
  - name: "reviews-v1-route"      #默认情况下请求转发到此路径
    route:
    - destination:
        host: reviews.prod.svc.cluster.local
        subset: v1

2.服务配置跨域

apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: ratings-route
spec:
  hosts:
  - ratings.prod.svc.cluster.local
  http:
  - route:
    - destination:
        host: ratings.prod.svc.cluster.local
        subset: v1
    corsPolicy:
      allowOrigins:
      - exact: example.com           #允许跨域的域名
      allowMethods:                  #允许跨域的请求
      - POST
      - GET
      allowCredentials: false
      allowHeaders:                  #允许的Headers
      - X-Foo-Bar
      maxAge: "24h"                  #有效期
配置 类型 描述 必需的
allowOrigin string[] 允许执行 CORS 请求的源列表。内容将被序列化到 Access-Control-Allow-Origin 标头中。通配符 * 将允许所有来源。 不
allowMethods string[] 允许访问资源的 HTTP 方法列表。内容将被序列化到 Access-Control-Allow-Methods 标头中。 不
allowHeaders string[] 请求资源时可以使用的 HTTP 标头列表。序列化为 Access-Control-Allow-Headers 标头。 不
exposeHeaders string[] 允许浏览器访问的 HTTP 标头白名单。序列化为 Access-Control-Expose-Headers 标头。 不
maxAge Duration 指定预检请求的结果可以缓存多长时间。翻译为Access-Control-Max-Age标题。 不
allowCredentials BoolValue 指示是否允许调用者使用凭据发送实际请求（而不是预检）。翻译为 Access-Control-Allow-Credentials标题。 不

3.配置路径转发

HttpMatchRequest 指定要满足的一组标准，以便将规则应用于 HTTP 请求。例如，以下内容将规则限制为仅匹配 URL 路径以 /ratings/v2/ 开头的请求，并且请求包含end-user带有 value 的自定义标头jason

apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: ratings-route
spec:
  hosts:
  - ratings.prod.svc.cluster.local
  http:
  - match:
    - headers:             #匹配headers
        end-user:
          exact: jason
      uri:
        prefix: "/ratings/v2/"     #匹配路径
      ignoreUriCase: true
    route:
    - destination:
        host: ratings.prod.svc.cluster.local
场地 类型 描述 必需的
name string 分配给匹配项的名称。匹配的名称将与父路由的名称连接，并将记录在与该路由匹配的请求的访问日志中。 不
uri StringMatch 匹配值的 URI 区分大小写，格式如下：exact: "value"用于精确的字符串匹配prefix: "value"用于基于前缀的匹配regex: "value"用于 ECMAscript 样式的基于正则表达式的匹配注意：ignore_uri_case可以通过标志启用不区分大小写的匹配 。 不
scheme StringMatch URI Scheme 值区分大小写，格式如下：exact: "value"用于精确的字符串匹配prefix: "value"用于基于前缀的匹配regex: "value"用于 ECMAscript 样式的基于正则表达式的匹配 不
method StringMatch HTTP 方法值区分大小写，格式如下：exact: "value"用于精确的字符串匹配prefix: "value"用于基于前缀的匹配regex: "value"用于 ECMAscript 样式的基于正则表达式的匹配 不
authority StringMatch HTTP 授权值区分大小写，格式如下：exact: "value"用于精确的字符串匹配prefix: "value"用于基于前缀的匹配regex: "value"用于 ECMAscript 样式的基于正则表达式的匹配 不
headers map<string, StringMatch> 标题键必须为小写并使用连字符作为分隔符，例如x-request-id。标头值区分大小写，格式如下：exact: "value"用于精确的字符串匹配prefix: "value"用于基于前缀的匹配regex: "value"用于 ECMAscript 样式的基于正则表达式的匹配注意：uri、scheme、method和键authority将被忽略。 不
port uint32 指定正在寻址的主机上的端口。许多服务只公开单个端口或使用它们支持的协议标记端口，在这些情况下，不需要显式选择端口。 不
sourceLabels map<string, string> 一个或多个标签，用于限制规则对具有给定标签的工作负载的适用性。如果 VirtualService 在顶部指定了网关列表，则它必须包含保留的网关 mesh以使该字段适用。 不
queryParams map<string, StringMatch> 查询参数进行匹配。例如： - 对于像“?key=true”这样的查询参数，映射键将是“key”，字符串匹配可以定义为exact: "true"。- 对于像“?key”这样的查询参数，映射键将是“key”，字符串匹配可以定义为exact: ""。- 对于像“?key=123”这样的查询参数，映射键将是“key”，字符串匹配可以定义为regex: "\d+$"。请注意，此配置仅匹配“123”之类的值，而不匹配“a123”或“123a”。注意： prefix目前不支持匹配。 不
ignoreUriCase bool 用于指定 URI 匹配是否不区分大小写的标志。注意：exact只有在和prefix URI 匹配的情况下才会忽略大小写。 不

4.配置301重定向

HTTPRedirect 可用于向调用者发送 301 重定向响应，其中 Authority/Host 和响应中的 URI 可以与指定的值交换。例如，以下规则将评级服务上的 /v1/getProductRatings API 请求重定向到 bookratings 服务提供的 /v1/bookRatings

apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: ratings-route
spec:
  hosts:
  - ratings.prod.svc.cluster.local
  http:
  - match:
    - uri:
        exact: /v1/getProductRatings
    redirect:
      uri: /v1/bookRatings
      authority: newratings.default.svc.cluster.local
场地 类型 描述 必需的
uri string 在重定向时，使用此值覆盖 URL 的路径部分。请注意，无论请求 URI 是否匹配为确切路径或前缀，都将替换整个路径。 不
authority string 在重定向时，使用此值覆盖 URL 的授权/主机部分。 不
redirectCode uint32 在重定向时，指定要在重定向响应中使用的 HTTP 状态代码。默认响应代码为 MOVED_PERMANENTLY (301)。 不

5.配置HTTP重试

描述 HTTP 请求失败时要使用的重试策略。例如，以下规则将调用 rating:v1 服务时的最大重试次数设置为 3，每次重试尝试超时 2 秒。

apiVersion: networking.istio.io/v1alpha3
kind: VirtualService
metadata:
  name: ratings-route
spec:
  hosts:
  - ratings.prod.svc.cluster.local
  http:
  - route:
    - destination:
        host: ratings.prod.svc.cluster.local
        subset: v1
    retries:
      attempts: 3
      perTryTimeout: 2s
      retryOn: gateway-error,connect-failure,refused-stream
场地 类型 描述 必需的
attempts int32 给定请求的重试次数。重试间隔将自动确定（25ms+）。实际重试次数取决于 httpReqTimeout。 是的
perTryTimeout Duration 给定请求的每次重试超时。格式：1h/1m/1s/1ms。必须 >=1 毫秒。 不
retryOn string 指定重试发生的条件。可以使用“,”分隔列表指定一个或多个策略。有关更多详细信息，请参阅重试策略 和gRPC 重试策略。

6.HTTP路由

场地 类型 描述 必需的
name string 为调试目的分配给路由的名称。路由的名称将与匹配的名称连接，并将记录在访问日志中以匹配此路由/匹配的请求。 不
match HTTPMatchRequest[] 激活规则需要满足的匹配条件。单个匹配块内的所有条件都具有 AND 语义，而匹配块列表具有 OR 语义。如果任何一个匹配块成功，则匹配该规则。 不
route HTTPRouteDestination[] http 规则可以重定向或转发（默认）流量。转发目标可以是服务的多个版本之一（参见文档开头的词汇表）。与服务版本相关的权重决定了它接收的流量比例。 不
redirect HTTPRedirect http 规则可以重定向或转发（默认）流量。如果在规则中指定了流量直通选项，则路由/重定向将被忽略。重定向原语可用于将 HTTP 301 重定向发送到不同的 URI 或授权。 不
rewrite HTTPRewrite 重写 HTTP URI 和授权标头。Rewrite 不能与 Redirect 原语一起使用。转发前会进行重写。 不
timeout Duration HTTP 请求超时。 不
retries HTTPRetry HTTP 请求的重试策略。 不
fault HTTPFaultInjection 故障注入策略应用于客户端的 HTTP 流量。请注意，在客户端启用故障时，将不会启用超时或重试。 不
mirror Destination 除了将请求转发到预期目标之外，还将 HTTP 流量镜像到另一个目标。镜像流量是在尽力而为的基础上，边车/网关在从原始目的地返回响应之前不会等待镜像集群响应。将为镜像目的地生成统计信息。 不
mirrorPercent UInt32Value 由mirror字段镜像的流量百分比。如果没有这个字段，所有的流量（100%）都会被镜像。最大值为 100。 不
corsPolicy CorsPolicy 跨域资源共享策略 (CORS)。有关跨源资源共享的更多详细信息，请参阅 CORS 。 不
headers Headers 标头操作规则 不
